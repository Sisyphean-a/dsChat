## 项目需求

本项目旨在开发一款轻量、优雅的 uTools 本地 AI 对话插件。核心能力包括：调用 DeepSeek API 进行流式对话、侧边栏历史记录管理、使用 `utools.db` 持久化用户配置（API Key/BaseURL 等）与聊天记录。特色功能为基于 uTools 生命周期的“1 分钟内重开恢复对话，超时新建对话”机制，且需原生支持 Markdown 解析与代码高亮，最终打包为 `.upx` 独立插件文件。

## 阶段 4：生命周期管理与“1分钟”重置机制

*(本阶段目标：利用 uTools 的底层进出事件，实现产品核心特性——基于时间窗的上下文保持逻辑。)*

### 🟢 Quest 4.1：监听进退场事件与时间戳打点

- **🎯 关卡目标**：利用生命周期钩子，在用户隐藏插件时记录当前系统时间；在用户再次呼出插件时，计算时间差并打印到控制台。
- **🛠️ 推荐工具/轮子**：`window.utools.onPluginEnter`、`window.utools.onPluginOut`、`Date.now()`。
- **🚫 边界限制 (防超纲)**：仅仅打印时间差数字，本关严禁修改 UI 或清空对话。
- **✅ 考官验收标准 (PM Checklist)**：
  - [ ] 测试动作 1：打开 uTools 开发者工具的控制台 (Console)。
  - [ ] 测试动作 2：隐藏插件（按 ESC），等待几秒钟后重新唤起插件。
  - [ ] 预期结果 1：控制台打印出类似于 `距离上次离开已过：5800 毫秒` 的日志。

### 🟢 Quest 4.2：实现 1 分钟上下文熔断逻辑

- **🎯 关卡目标**：依据上一关计算出的时间差。如果差值 > 60000 毫秒（1分钟），清空当前 Vue 状态中的对话数组；否则，保持原样。
- **🛠️ 推荐工具/轮子**：Vue 响应式状态管理。
- **🚫 边界限制 (防超纲)**：不需要管历史记录列表如何更新，只需要清空主聊天区域的数据流即可。
- **✅ 考官验收标准 (PM Checklist)**：
  - [ ] 测试动作 1：在输入框发一条假消息（出现在屏幕上），关闭插件，10秒后打开。
  - [ ] 预期结果 1：刚刚发的消息还在界面上。
  - [ ] 测试动作 2：关闭插件，使用手机计时 61 秒后重新打开插件。
  - [ ] 预期结果 2：主界面变成干净的新对话状态，刚才的消息消失。

## 阶段 5：DeepSeek API 握手与流式打字机效果

*(本阶段目标：攻克核心难点——使用配置好的 Key 和 Model 向大模型发请求，并解析服务器发送事件 (SSE) 实现打字机效果。)*

### 🟢 Quest 5.1：联通 DeepSeek 并获取全量回复

- **🎯 关卡目标**：提取设置中的 API Key 和模型名，点击发送按钮时，向 DeepSeek 发送简单的 POST 请求（`stream: false`），把返回结果渲染到气泡。
- **🛠️ 推荐工具/轮子**：原生的 `fetch` 函数。
- **🚫 边界限制 (防超纲)**：严禁使用 `stream: true`。本关只求走通网络链路，不要关心等待时的 Loading 状态。
- **✅ 考官验收标准 (PM Checklist)**：
  - [ ] 测试动作 1：配置真实的 API Key，在输入框输入“你是谁”并发送。
  - [ ] 预期结果 1：停顿几秒后，界面突然直接出现一段完整的回答（如“我是 DeepSeek...”）。

### 🟢 Quest 5.2：升级为 SSE 流式渲染 (打字机效果)

- **🎯 关卡目标**：将请求参数改为 `stream: true`，使用流读取器逐块（chunk）解析 DeepSeek 的返回流，并实时更新到 Vue 气泡状态中。
- **🛠️ 推荐工具/轮子**：`response.body.getReader()`、`TextDecoder`、或者更高级的 npm 包 `@microsoft/fetch-event-source` 避免手写解析逻辑。
- **🚫 边界限制 (防超纲)**：目前流出来的必定是带有 Markdown 符号的纯文本，不要尝试在本关将其变成漂亮的排版格式。
- **✅ 考官验收标准 (PM Checklist)**：
  - [ ] 测试动作 1：输入框输入“讲一个 100 字的故事”。
  - [ ] 预期结果 1：气泡框内立刻开始按字/词级别连续浮现文本（像人打字一样），而不是等待很久后一次性全部蹦出来。

### 🟢 Quest 6.2：接入 highlight.js 赋予代码生命力

- **🚫 边界限制 (防超纲)**：本关暂不需要实现代码块右上角的“一键复制”按钮。

  - [ ] 预期结果 1：AI 返回的代码块具有深色/浅色背景，并且代码关键字（如 `def`, `return`）有不同的颜色高亮区分。
